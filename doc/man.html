Waiting4Godot

In its first release, Waiting4Godot (W4G) is an UML modeling tool designed with the purpose of producing business analyzes of software systems that are intuitive to make and agree upon.

Hence, it also embodies an approach yet flexible to allow for the different The approach is influenced by than the original Grady, Allen & Boosche ...

The model

Using the tool

Diagrams

Use Case Diagram

Supports the following elements: Actor, Use Case, etc.

Control Level

Use Case Diagrams are not constrained.

Grouping Elements

Use Packages to group Use Cases & Actors.

Objects

Objects identified while constructing the diagram may be captured as follows:

  • Business Objects can be captured in Use Case elements.
  • Controllers can be captured in Actor elements.

Its also possible to link Objects captured in other elements.

Attached Diagrams

Its possible to catalog further analysis of certain Elements by creating Attached Diagrams. Elements created in these Attached Diagrams are available for use elsewhere in the model. The following Elements support Attached Diagrams.

  • Use Case

Activity Diagram

Supports the following elements: Activity (Action, Call Activity, Accept Event, Accept Time Event, Send Signal) , Activity Node (Start, Stop,Flow End) & Control Node (Conditional Block, While Loop & Repeat Loop).

Control Level

Use Case Diagrams may be free or constrained.

  • Free: Suitable mode for high level analysis to help identify and capture Objects, properties & methods.
  • Constrained. Only methods of Objects captured in other diagrams can be used as Activities.
Grouping Elements

Use swimlanes to group Use Cases & Actors.
